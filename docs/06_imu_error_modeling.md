# Noisy IMU Modeling and INS Error Propagation

## Overview

Real inertial navigation systems do not operate with ideal sensors.

Accelerometers and gyroscopes contain multiple stochastic error sources that
accumulate over time and eventually degrade the navigation solution.

The objective of this experiment is twofold:

1. Model realistic inertial sensor noise processes.
2. Investigate how those sensor errors propagate through a strapdown inertial navigation system.

The experiment begins by generating Monte Carlo realizations of several common IMU error sources and validating their statistical properties using Allan variance analysis.

These noise processes are then injected into the IMU measurements generated by the inverse mechanization experiment, and the resulting navigation errors are studied using forward mechanization.

This experiment bridges the gap between ideal inertial navigation and realistic inertial navigation.

---

# Background

## Why Sensor Noise Matters

An inertial navigation system estimates:

- position,
- velocity,
- attitude,

by integrating accelerometer and gyroscope measurements.

Because integration accumulates errors, even very small sensor imperfections can produce large navigation errors over time.

The effect is especially severe for:

- gyroscope errors, which corrupt attitude estimates,
- and accelerometer errors, which are integrated twice into position.

Understanding these error sources is one of the most important aspects of inertial navigation system design.

---

# IMU Error Sources

Five common stochastic IMU error processes are modeled.

---

## 1. Quantization Noise

Quantization noise results from finite sensor resolution.

The sensor output can only represent discrete values, causing small rounding errors in the measurements.

Characteristics:

- very high-frequency noise,
- appears white,
- usually the smallest contributor.

---

## 2. White Noise

White noise represents random measurement uncertainty that is independent from sample to sample.

Characteristics:

- zero mean,
- uncorrelated samples,
- constant power spectral density.

White noise is often the dominant short-term sensor error.

---

## 3. Bias Instability (FOGM)

Bias instability is modeled as a first-order Gauss-Markov (FOGM) process.

Unlike white noise, bias instability is correlated over time and behaves like a slowly varying sensor bias.

Characteristics:

- low-frequency behavior,
- finite correlation time,
- often one of the most important practical error sources.

---

## 4. Brownian Noise (Rate Random Walk)

Brownian noise represents the integral of white noise.

Characteristics:

- random walk behavior,
- variance grows with time,
- causes long-term drift.

---

## 5. Ramp Noise

Ramp noise represents a continuously drifting sensor bias.

Characteristics:

- very low-frequency behavior,
- error growth faster than Brownian motion,
- often dominates at extremely long averaging times.

---

# Allan Variance Analysis

## Motivation

Individual noise realizations can be difficult to interpret directly.

Allan variance provides a powerful tool for identifying and estimating the underlying stochastic error sources present in an inertial sensor.

Different noise processes produce characteristic slopes in the Allan variance curve.

---

## Allan Variance Calculation

The five noise processes are combined to create a composite sensor error signal.

For each Monte Carlo realization:

1. Generate composite noise.
2. Compute Allan variance.
3. Average the resulting Allan variance curves.

A least-squares fit is then performed to estimate the original noise parameters.

---

## Allan Variance Results

![Allan Variance](../results/figures/noisy_imu_demo/allan_variance_fit.pdf)

The figure shows:

- individual Monte Carlo Allan variances,
- mean Allan variance,
- ideal Allan variance,
- fitted Allan variance model.

---

## Observation

The fitted model closely matches the ideal Allan variance curve.

The estimated noise parameters are consistent with the original simulation parameters, demonstrating that the Allan variance implementation successfully recovers the underlying sensor characteristics.

# Part II: Noisy Flight Simulation

## Motivation

The previous examples used ideal IMU measurements.

The next question is:

> How do realistic sensor errors affect the navigation solution?

To answer this question, the synthetic IMU measurements generated by inverse mechanization are corrupted with the previously modeled sensor noise.

The resulting noisy measurements are then propagated through the forward mechanization algorithm.

---

# Experimental Setup

The aircraft follows the same figure-eight trajectory used in previous experiments.

Noise is added to selected IMU channels before forward mechanization.

Ten Monte Carlo simulations are performed for each case.

Altitude aiding is enabled throughout the simulation using ideal height measurements.

Conceptually, this represents an ideal barometric altimeter.

The altitude aiding prevents vertical divergence while still allowing sensor noise to influence the navigation solution.

---

# Noise Injection Cases

The following scenarios are evaluated:

| Case | Noise Source |
|--------|-------------|
| 1 | Gyroscope x, y, z |
| 2 | Gyroscope x |
| 3 | Gyroscope y |
| 4 | Gyroscope z |
| 5 | Accelerometer x, y, z |
| 6 | Accelerometer x |
| 7 | Accelerometer y |
| 8 | Accelerometer z |
| 9 | All accelerometer and gyroscope axes |

For visualization purposes, accelerometer noise is scaled to make its effects more apparent.

# Key Insights

## Why Gyroscope Errors Are So Dangerous

Accelerometer measurements are transformed into the navigation frame using the estimated attitude.

An attitude error causes every subsequent acceleration measurement to be projected incorrectly.

As a result:

```text
Gyro Error
    ↓
Attitude Error
    ↓
Acceleration Error
    ↓
Velocity Error
    ↓
Position Error
```

This cascading effect explains why gyroscope quality is often more important than accelerometer quality in many navigation systems.

---

## Role of Altitude Aiding

The forward mechanization includes ideal altitude aiding throughout the simulation.

This prevents the vertical channel from becoming unstable and allows the experiment to focus on horizontal navigation error growth.

The aiding serves as a simplified model of:

- barometric altitude measurements,
- radar altimeters,
- or other external vertical references.

---

# Key Takeaways

- Real IMUs contain multiple stochastic error sources.
- Allan variance can be used to identify and estimate those error sources.
- Bias instability and random walk processes dominate long-term performance.
- Gyroscope errors typically produce larger navigation errors than accelerometer errors.
- Attitude errors are particularly dangerous because they corrupt every subsequent acceleration measurement.
- Even small sensor imperfections eventually produce significant navigation drift.
- External aiding sources are often required to bound inertial navigation errors over long durations.

This experiment marks the transition from ideal inertial navigation to realistic inertial navigation and motivates the need for integrated navigation systems such as GPS/INS fusion.
